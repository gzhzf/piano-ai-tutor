# 🎹 琴乐启蒙AI导师 - 腾讯云部署指南

## 完整步骤（从注册到上线）

---

## 第一阶段：注册与认证（约1-2个工作日）

### 1. 注册腾讯云账号
1. 打开 https://cloud.tencent.com/register
2. 用手机号或微信扫码注册
3. 完成手机验证

### 2. 实名认证（必须）
1. 登录后，点击右上角头像 → **实名认证**
2. 选择个人认证
3. 上传身份证正反面 + 人脸识别
4. 等待审核（通常1-2小时，最慢1个工作日）

> ⚠️ 没有实名认证无法购买任何云服务器

---

## 第二阶段：购买轻量应用服务器（约10分钟）

### 3. 购买服务器
1. 打开 https://console.cloud.tencent.com/lighthouse
2. 点击 **新建** 或 **购买**
3. 按以下配置选择：

| 配置项 | 选择 |
|--------|------|
| **地域** | 广州 / 上海 / 北京（选离你最近的） |
| **镜像** | Ubuntu 22.04 LTS（系统镜像） |
| **套餐** | 最低套餐即可（2核2G，足够跑Flask） |
| **时长** | 按需选择（学生优惠约50元/年） |

4. 设置登录方式：选择 **使用密钥** 或 **使用密码**
   - 推荐选密码，设置一个你记得住的服务器密码
5. 点击 **购买** → 完成支付

### 4. 查看服务器公网IP
购买成功后，在轻量应用服务器列表中可以看到：
- **公网IP**：类似 `43.xxx.xxx.xxx`（记下来！）
- 这就是你智能体的公网地址

---

## 第三阶段：一键部署（约5分钟）

### 5. 登录服务器
在腾讯云控制台点击服务器 → **登录** → 选择 **标准登录方式**
输入你设置的密码，进入终端。

### 6. 运行部署脚本
在终端中粘贴执行以下**一条命令**：

```bash
bash <(curl -s https://raw.githubusercontent.com/gzhzf/piano-ai-tutor/main/deploy.sh)
```

> 如果上面命令失败，用以下手动方式：
```bash
apt-get update && apt-get install -y git python3 python3-pip python3-venv nginx
git clone https://github.com/gzhzf/piano-ai-tutor.git /opt/piano-ai
cd /opt/piano-ai
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
# 然后手动创建服务（见下方说明）
```

### 7. 开放防火墙端口
1. 回到腾讯云控制台 → 轻量应用服务器 → 点击你的服务器
2. 找到 **防火墙** 标签
3. 点击 **添加规则**：
   - 应用类型：HTTP (80)
   - 协议：TCP
   - 端口：80
   - 策略：允许
4. 保存

### 8. 验证访问
打开浏览器，访问：
```
http://你的公网IP
```
看到智能体界面即部署成功！

---

## 第四阶段（可选）：配置域名

如果需要用域名访问（如 `piano.yourdomain.com`）：
1. 在腾讯云购买域名（或转入已有域名）
2. 域名解析 → 添加A记录，指向服务器公网IP
3. 在服务器上配置SSL证书（免费证书可在腾讯云申请）

## 常见问题

### Q: 部署脚本运行失败？
A: 逐步排查：
```bash
# 查看服务状态
systemctl status piano-ai
# 查看错误日志
journalctl -u piano-ai -f
# 手动启动测试
cd /opt/piano-ai && source venv/bin/activate
python3 app.py
```

### Q: 访问不了？
A: 检查两点：
1. 腾讯云防火墙是否开放了80端口
2. 服务器内部ufw是否放行：`ufw allow 80/tcp`

### Q: 如何更新代码？
A: SSH登录服务器后：
```bash
cd /opt/piano-ai
git pull origin main
systemctl restart piano-ai
```

### Q: 如何重启服务？
```bash
systemctl restart piano-ai    # 重启Flask应用
systemctl restart nginx       # 重启Nginx
```
